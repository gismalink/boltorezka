## Prod fix: SSO redirect host mismatch (datowave)

- Симптом: `https://datowave.com/v1/auth/sso/start?...` возвращал на `https://auth.gismalink.art/...`.
- Причина: в server env `~/srv/boltorezka/infra/.env.host` оставались legacy значения:
  - `PROD_CORS_ORIGIN=https://boltorezka.gismalink.art`
  - `PROD_AUTH_SSO_BASE_URL=https://auth.gismalink.art`
  - `PROD_ALLOWED_RETURN_HOSTS=boltorezka.gismalink.art`
- Действия:
  - значения обновлены на `datowave.com` / `auth.datowave.com`;
  - выполнен штатный prod deploy API через `npm run deploy:prod` (ref `origin/main`), контейнер `boltorezka-api-prod` пересоздан.
- Проверка:
  - `https://datowave.com/v1/auth/sso/start?provider=google...` -> `302 Location: https://auth.datowave.com/auth/google...`
  - `https://datowave.com/v1/auth/sso/start?provider=yandex...` -> `302 Location: https://auth.datowave.com/auth/yandex...`
  - `Access-Control-Allow-Origin: https://datowave.com`.
