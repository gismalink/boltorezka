## Signed RC cycle execution (test channel)

### Summary

- Signed RC cycle запущен через GitHub manual workflow на `origin/main`.
- Cycle завершился `FAIL` на обеих platform matrix jobs в шаге signed build.

### Run evidence

- Workflow run:
  - `https://github.com/gismalink/datowave/actions/runs/23116286958`
- Inputs:
  - `release_channel=test`
  - `signed=true`
  - `create_release_draft=false`

### Root cause

- В CI run отсутствовали required signing/notarization env values (пустые secrets):
  - `CSC_LINK`, `CSC_KEY_PASSWORD`, `APPLE_ID`, `APPLE_APP_SPECIFIC_PASSWORD`, `APPLE_TEAM_ID`, `WIN_CSC_LINK`, `WIN_CSC_KEY_PASSWORD`.
- Подтверждение через repo secrets list:
  - команда `gh secret list --repo gismalink/datowave | rg "DESKTOP_(...)"` не вернула required keys.

### Decision

- Signing readiness gate remains `blocked` до заполнения required secrets и повторного signed RC cycle.

## Windows signing via Azure OIDC (transition path)

### Summary

- Внедрен переходный flow: Windows signed RC можно выполнять без Windows PFX secrets через Azure Trusted Signing (OIDC).
- Добавлен режим `windows-only` для signed dispatch, чтобы macOS не блокировал цикл до появления Apple developer credentials.

### Implementation

- Workflow updates:
  - `.github/workflows/desktop-artifacts.yml`
  - new dispatch inputs:
    - `signed_platforms = all | windows-only | mac-only`
    - `windows_signing_provider = azure-oidc | pfx`
  - `azure-oidc` path uses:
    - `azure/login@v2`
    - `azure/artifact-signing-action@v1`
- Runbook updates:
  - `docs/runbooks/DESKTOP_SIGNING_READINESS_RUNBOOK.md`
  - added Azure Trusted Signing secrets template and practical launch mode.

### Practical launch mode (current)

- Run workflow with:
  - `signed=true`
  - `signed_platforms=windows-only`
  - `windows_signing_provider=azure-oidc`
- Expected result:
  - Windows signed artifacts path active,
  - macOS unsigned path (non-blocking until Apple account onboarding complete).
