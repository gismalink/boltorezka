## Chat transport: WSS-first send model completion (topic/reply/room)

- Branch: `feature/chat-date-separator`.
- Goal: finish target `chat.send` transport model with WS-first and transient HTTP fallback across all send paths.

### What changed

- Web transport command layer:
  - Added `runChatSend` with policy `ws-first-http-fallback` for `chat.send`.
  - Topic send and reply send now use unified WS-first command path with fallback.
- Web send service:
  - Topic/reply send switched from direct HTTP to `runChatSend`.
  - Room text send keeps optimistic WS enqueue path (`chatController.sendMessage`), and now has fallback through `runChatSend` when direct enqueue is unavailable.
- API realtime handler (`chat.send`):
  - Added topic/reply handling via `room-topic-messages-service` in WS path.
  - Preserved inbox notifications in WS topic/reply flow (`emitReplyInboxEvent`, `emitMentionInboxEvents`).
- API HTTP fallback for room send:
  - Added `POST /v1/rooms/:slug/messages` to support room text fallback when WS transport is temporarily unavailable.
  - Endpoint includes access, NSFW, muted, readonly, and slowmode enforcement and emits realtime `chat.message` envelope.
