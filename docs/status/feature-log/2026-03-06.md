## 2026-03-06 — RTC Phase 2: negotiation manager + fairness queue

### Delivered

- В `apps/web/src/hooks/rtc/voiceCallNegotiationState.ts` добавлен централизованный queue-state manager:
	- `createOfferQueueState`,
	- `enqueueOfferRequest` (dedupe + bucket priority),
	- `dequeueNextOfferRequest` (fair round-robin by bucket),
	- `markOfferQueueActiveForTarget`,
	- `clearOfferQueue`.
- В `apps/web/src/hooks/rtc/voiceCallOfferPolicy.ts` добавлены retry policy primitives:
	- `resolveOfferRetryBudget`,
	- `resolveOfferRetryDelayMs`,
	- bucket-specific budget/delay для `manual|ice-restart|video-sync`.
- В `apps/web/src/hooks/rtc/useVoiceCallRuntime.ts` `startOffer` переведён на queue-first flow:
	- enqueue вместо прямого выполнения,
	- single drain loop,
	- retry по budget при preflight/signal failures,
	- сохранена текущая `none` default-политика для video effects.

### Validation

- `npm --prefix apps/web run -s build` — PASS.

### Decision

- Phase 2 (`negotiation reliability`) переведен в DONE.
- Следующий активный этап: `#6.3` (observability + smoke/deploy gates).
