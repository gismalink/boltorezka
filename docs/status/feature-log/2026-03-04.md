- `SMOKE_API_URL=https://test.datowave.com SMOKE_WEB_BASE_URL=https://test.datowave.com SMOKE_BEARER_TOKEN=<token> SMOKE_E2E_CALL_SIGNAL=0 SMOKE_E2E_RECONNECT=0 npm run smoke:web:e2e` — PASS.
- `SMOKE_API_URL=https://test.datowave.com SMOKE_BEARER_TOKEN=<token> SMOKE_CALL_SIGNAL=1 SMOKE_RECONNECT=0 npm run smoke:realtime` — FAIL-fast с явной причиной same-user ticket pair.
- `scripts/smoke-web-denied-media.mjs` обновлён для совместимости с текущими camera icon классами (`bi-camera-video*`).

### Smoke root cause + hardening

- Root cause call-signal timeout: два ws-ticket выдавались для одного и того же userId; второй join в non-text channel эвиктил первый socket (`ChannelSessionMoved`).
- В `scripts/smoke-web-e2e.sh` auto-ticket генерация обновлена: `SMOKE_WS_TICKET_SECOND` берётся из другого пользователя (`SMOKE_USER_EMAIL_SECOND` или авто-подбор `email <> SMOKE_USER_EMAIL`).
- В `scripts/smoke-realtime.mjs` добавлен fail-fast guard на same-user first/second ticket pair с явным сообщением.

### Decision

- Hotfix внедрён и локально подтверждён по доступному browser smoke.
- Full-default e2e gate требует отдельной стабилизации call-signal stage (`call.offer` timeout).
