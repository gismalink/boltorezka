## 2026-03-04 — Web video runtime and server controls (sender-side effects + ASCII)

### Delivered

- Реализован sender-side video effects pipeline для outgoing camera track:
  - `none` (raw),
  - `pixel8` (8-bit quantization + dithering + optional grid),
  - `ascii` (glyph-based luminance rendering).
- Добавлен server-level selector типа эффекта в `Server -> Video` (`dropdown`) и условный рендер соответствующих настроек.
- Добавлены и интегрированы настройки `ASCII`:
  - glyph size,
  - contrast,
  - font color.
- Добавлен owner preview в начале Server Video tab; preview использует те же параметры обработки, что и outgoing pipeline.
- Video windows overlay доработан:
  - drag за любую область окна,
  - resize за любой угол,
  - resize handles видимы только при hover/focus,
  - local/self tile mirrored по X.
- Server controls расширены для ресайза окон:
  - min width,
  - max width (увеличен до `480`).

### Validation

- Повторные локальные `apps/web -> npm run build` — PASS на каждом инкременте.
- Feature test rollouts выполнены последовательно и зафиксированы smoke PASS:
  - `1dea7b3` — owner preview + sender-side pipeline,
  - `e16dacd` — video window UX + pixel-grid control,
  - `fd93196` — ASCII effect + effect type selector,
  - `38588f4` — server controls: resize bounds + ASCII color,
  - `1c40a14` — compact sliders layout + max resize width `480`.
- Последний зафиксированный прогон:
  - `ssh mac-mini 'cd ~/srv/boltorezka && TEST_REF=origin/feature/video-stream-overlay-chat-toggle npm run deploy:test:smoke'` — PASS (`health`, `smoke:sso`, `smoke:api`, `smoke:realtime`).

## 2026-03-04 — Runtime auto-update of input device on system devicechange

### Delivered

- Для active call добавлена runtime-реакция на `navigator.mediaDevices.devicechange` в media-effects layer.
- При system switch устройства ввода выполняется авто-refresh исходящего audio track:
  - работает для explicit device id,
  - работает для `selectedInputId=default` (частый кейс macOS auto-route при смене наушников).
- В call log добавлены явные записи о success/failure авто-обновления input track.

### Validation

- `npm run check:api-types` — PASS.

### Decision

- Пункт roadmap по авто-обновлению input device при system devicechange переведён в **DONE**.
