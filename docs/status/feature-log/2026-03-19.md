# 2026-03-19

## Prod RTC incident: relay-only policy leaked to production

### Summary

- После rollout в `prod` WebRTC-сессии начали падать с `VC-ICE-001`.
- Корневая причина: в `infra/.env.host` для production был выставлен `PROD_VITE_RTC_ICE_TRANSPORT_POLICY=relay`.
- Это привело к relay-only поведению на клиентах и повторяемым ICE fail в LiveKit (`responsesReceived=0`).

### Evidence

- API telemetry в `prod`: `rtc_connect_failed`, `code=VC-ICE-001`, `category=ICE_CONNECTIVITY`.
- LiveKit `prod`: `removing participant without connection`, `connectionType=unknown`, `ICE candidate pair stats ... state=failed`, `responsesReceived=0`.

### Fix applied

- Исправлено значение в `infra/.env.host`:
  - `PROD_VITE_RTC_ICE_TRANSPORT_POLICY=all`
- Выполнен повторный production deploy из `origin/main`.
- Подтверждено в собранном bundle: `VITE_RTC_ICE_TRANSPORT_POLICY:"all"`.
- `/health` вернул `status=ok`, checks `api/db/redis=ok`.

### Prevention

- Добавлен safety gate в `scripts/deploy/deploy-prod-from-ref.sh`:
  - если `PROD_VITE_RTC_ICE_TRANSPORT_POLICY=relay`, deploy в `prod` блокируется;
  - для осознанного обхода требуется явный override: `ALLOW_PROD_RELAY_ONLY=1`.

### Notes

- Миграции проверены отдельно: `schema_migrations` содержит `0001..0004` в `prod`, `npm run migrate` идемпотентен.

## TURN2 verification follow-up (native baseline)

### Summary

- Проверка TURN2 переведена на актуальный baseline: `turn2.gismalink.art -> 46.149.71.86`, native `coturn` (systemd), relay range `49160-49359` (200 портов).
- Прямой серверный TURN auth/allocate тест (`turnutils_uclient`) подтверждает рабочий путь `turns:443`.
- Browser smoke `run-realtime-media-test-room.sh` при `SMOKE_ALLOW_LEGACY_CALL_SIGNAL=1` падает в стадии signaling (`offer/answer` не релеятся), что не является доказательством отказа TURN2.

### Changes applied

- Обновлён runbook: `docs/runbooks/TURN2_VPS_SETUP_RUNBOOK.md` (native flow, текущий IP, текущий портовый диапазон).
- Обновлён release checklist: `docs/runbooks/workflow-checklist.md` (убран устаревший extended relay gate на legacy call signaling, добавлены LiveKit проверки).
- Добавлен fail-fast guard в `scripts/smoke/run-realtime-media-test-room.sh`:
  - без `SMOKE_ALLOW_LEGACY_CALL_SIGNAL=1` скрипт теперь завершается с ошибкой,
  - это предотвращает ложный PASS в режиме `skipped`.

### Operational note

- Для валидации именно TURN2 (без зависимости от legacy signaling path) использовать direct coturn probe:
  - `turnutils_uclient -S -T -p 443 -u <user> -w <pass> turn2.gismalink.art`.
