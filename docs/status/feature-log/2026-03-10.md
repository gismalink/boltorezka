# 2026-03-10

## Prod RTC incident fix: deprecated LiveKit path + LiveKit upgrade

### Incident

- After prod rollout, clients failed RTC connect with `v1 RTC path not found`.
- Browser requests showed deprecated path usage (`/livekit/rtc/*`).

### Root cause

- `PROD_LIVEKIT_URL` in server env was set to `wss://boltorezka.gismalink.art/livekit`.
- Current routing policy is livekit canonical path (`/rtc/*`), while `/livekit/rtc/*` is intentionally deprecated and returns `404`.

### Fix

- Updated server env value:
  - from: `PROD_LIVEKIT_URL=wss://boltorezka.gismalink.art/livekit`
  - to: `PROD_LIVEKIT_URL=wss://boltorezka.gismalink.art`
- Re-ran prod deploy from `origin/main`.
- Upgraded prod LiveKit container to `livekit/livekit-server:v1.9.12` (interactive keychain unlock + standard compose pull/recreate on server).

### Validation

- Route checks on prod:
  - `GET /livekit/rtc/validate` -> `404` (expected deprecated behavior)
  - `GET /rtc/validate` -> `401` (canonical path alive, auth required)
  - `POST /v1/auth/livekit-token` without auth -> `401` (endpoint alive/protected)
- Health check:
  - `GET /health` -> `ok` (`api/db/redis` all `ok`)
- LiveKit runtime:
  - Running image: `livekit/livekit-server:v1.9.12`
  - Startup log version: `1.9.12`
